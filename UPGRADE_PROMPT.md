## กฎเหล็ก (เพิ่มจาก CLAUDE.md เดิม — ทั้งหมดในนั้นยังใช้)

1. **ห้ามเดา ห้ามมั่ว** — เนื้อหายึด `transcripts/` + PDF ใน `เนื้อหาเรียน/` เท่านั้น ส่วนเสริมนอกแหล่งต้องแปะป้ายชัดว่าเป็นส่วนเสริม จุดที่เลคเชอร์กับสไลด์**ขัดกัน**ให้มาร์ก ⚠︎ พร้อมบอกว่ายึดอะไรเพราะอะไร
2. **ตัวเลขทุกตัวในบทต้องยืนยันด้วยโปรแกรมก่อนใส่** — เขียนสคริปต์คำนวณเช็ค (dB, กำลังงาน, link budget ฯลฯ) ห้ามคำนวณในหัวแล้วพิมพ์เลย
3. **สิ่งที่อาจารย์พูดเน้นในเลคเชอร์** (จาก transcript) ให้ใส่และติดป้าย 🎙️ — มุกตลก อุปมา คำเตือน "ตรงนี้ออกสอบ" คือของมีค่าที่สุด
4. เนื้อหาภาษาไทย ศัพท์เทคนิคอังกฤษ นิยามศัพท์ใหม่แบบ tiered (ง่ายก่อน → ลึก)

## โครงบทเรียนต่อ 1 week (ตามสูตร Numer Master — ครบทุกข้อ)

1. **ทำไมต้องเรียน + อุปมาโลกจริง** — เปิดด้วยสถานการณ์จริงจับต้องได้ (เช่น dB: "ทำไมวิศวกรไม่บวกเลขธรรมดา", link budget: "สัญญาณจากเสาถึงมือถือเหลือเท่าไหร่", packet switching: "ทำไมเน็ตไม่ล่มเวลาคนใช้พร้อมกัน")
2. **ทฤษฎีจากราก** — ไล่ที่มาทีละขั้น อธิบาย "ทำไม" ของทุกสูตร (log ทำไมเปลี่ยนคูณเป็นบวก → dB ทำไมมี 10 → dBm ต่างจาก dBW ตรงไหน) ห้ามโยนสูตรลอย ๆ
3. **Animation อย่างน้อย 4–6 ตัวต่อบท** — สเปกด้านล่าง
4. **ตัวอย่างทำมือทีละขั้น** — เดินโจทย์จริงจากชีท/เลคเชอร์แบบ step-by-step โชว์วิธีเขียนลงกระดาษคำตอบ
5. **Interactive widget** — เครื่องคิด/ทดลองที่ผู้ใช้ปรับค่าเองได้ (เช่น link budget calculator, dB converter, ตาราง encapsulation)
6. **วิธีคิดไว ๆ สำหรับห้องสอบ** — สูตรลัดคิด dB ในหัว (3 dB = ×2, 10 dB = ×10), งบเวลาต่อข้อ, ลำดับการกดเครื่องคิดเลข
7. **โค้ดรันได้** — Python/JS เล็ก ๆ คำนวณโจทย์เดียวกับที่ทำมือ ให้ผลตรงกัน
8. **ข้อสอบจำลองจับเวลา** — ทั้งเขียนมือและโจทย์ประยุกต์ อย่างน้อย 4–5 ข้อ มีตัวจับเวลา + เฉลยล็อกจนหมดเวลา (ดู `TimedExam` ใน `~/Downloads/numer/lib/components.jsx` เป็นต้นแบบ แล้ว port เป็นของโปรเจกต์นี้)
9. **Quick reference** — สรุปสูตร/แนวคิดท้ายบท 1 จอ + กับดักพลาดบ่อย

## สเปก Animation (หัวใจของงานนี้)

- SVG interactive มี step player (ปุ่ม เล่น/ถอยหลัง/เดินหน้า + คำอธิบายเปลี่ยนตามขั้น) — ไม่ใช่ gif ไม่ใช่ภาพนิ่ง
- 1 ตัว สอน 1 ไอเดีย ใช้**ตัวเลขจริงจากตัวอย่างในบท** (ให้ animation กับตัวอย่างทำมือเป็นเรื่องเดียวกัน)
- ไอเดียเฉพาะวิชานี้ที่ควรทำ: สเกล log/dB ที่เลื่อนได้ · link budget แบบ waterfall (กำลังงานไหลผ่าน Tx → cable loss → antenna gain → path loss → Rx) · packet ถูกห่อทีละชั้นแล้วแกะ (encapsulation ผ่าน TCP/IP 5 ชั้น) · เทียบ OSI ↔ TCP/IP แบบ mapping มีไฮไลท์ · circuit vs packet switching แข่งกันส่งข้อมูล · datagram vs virtual circuit (เส้นทางแพ็กเก็ต) · ผลของขนาดแพ็กเก็ตต่อเวลารวม
- ทุก section ที่มีข้อความติดกันยาวเกิน ~150 คำ ให้แปลงบางส่วนเป็นภาพ/ตาราง/ไทม์ไลน์/quiz
